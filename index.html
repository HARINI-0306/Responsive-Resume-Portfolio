01 · Work
Selected work

Two projects I actually finished (which, in student-land, counts for a lot).

MeterFlow

API Billing Platform
JavaSpring BootMySQLReact

I wanted to understand what actually happens behind a "usage-based billing" API — the kind SaaS products use to track and charge for API calls. So I built one myself: a platform that manages users, generates API keys, tracks usage, and turns that into billing data.

The backend runs on Java and Spring Boot with a proper layered structure — controllers, services, repositories — instead of dumping everything into one file like my earlier attempts. MySQL handles the users, keys, and usage logs.

What I got out of it: my first real sense of how backend architecture holds together when an app has more than one moving part — and why "just make it work" and "make it work cleanly" are very different problems.

Rini Web Studio

Portfolio & web development studio
HTMLCSSJavaScriptResponsive Design

This one started as a way to teach myself responsive layout properly, and turned into an actual small business. It's the site I use to show potential clients what a Rini Web Studio site looks like — services, past work, and a way to get in touch.

I paid close attention to things that are easy to skip as a beginner: layout that holds up on a phone, a homepage that loads fast, and a structure search engines can actually read.

What I got out of it: the gap between "a website that works on my laptop" and "a website that works for a client's actual customers" — and a growing client-facing habit of writing things simply.
